## 二、根因排查(已完成)

### 结论:连接弦不是轨道断裂,而是**惩罚豁免导致的"抄近路"**

逐项验证:

1. **轨道没断**。建一张只含铁轨边(去掉连接边)的图做连通分量分析:**全部 34 处连接弦的两端都在同一连通分量里**——铁轨是通的。

2. **连接弦≈平行的真实铁轨**。测每处连接弦两端的"纯铁轨最短路":
   - 27 处 ratio ≤1.5(铁轨和连接弦几乎一样长/平行),
   - 5 处是较大抄近路(郡山→福島 ×14.6、上田→長野 ×13.9、佐久平→上田 ×14.5、立野→赤水 ×5.6、札幌→新札幌 ×7.4),
   - 2 处铁轨绕行 >30km(仙台→大宮,连接弦纯属多余捷径)。

3. **A/B 实测**:把 18 班受影响列车在**禁用连接边**后重算 —— **全部 0 报错、区段数不变、总长度变化仅 −0.8%…+0.7%、无绕路**。最大跳变随之下降(やくも 860→656m、剣山 889→627m、宇和海 870→572m、しおかぜ 856→592m、にちりん 836→709m)。说明真实铁轨一直就在旁边、可走、几乎等长。

4. **机制锁定到一行代码** —— `app/public/app.js` 第 4215 行:

   ```js
   let weight = edge.length + (edge.is_station_connector ? 0 : institutionPreferencePenaltyForEdge(edge, allowedCodes, train));
   if (!edge.is_station_connector) {
     weight += nonPreferredLineOperatorPenalty(edge, ...);   // 4217
   }
   ```

   连接边**同时豁免了** institution 惩罚和 line/operator 惩罚,代价只剩 `gap + 180`。而在线名切换的走廊上(児島↔岡山 的 本四備讃/宇野線、伊予大洲↔内子 的 予讃旧/新线、旭川↔上川 的 石北本线 等),**平行的真实铁轨被判成"非偏好线/运营商",累加了惩罚**,于是无惩罚的连接弦反而更便宜 → Dijkstra 抄近路、画成一条偏离轨道的直线。`STATION_TRANSFER_MAX_NODE_GAP_METERS = 900` 决定了这些弦全部 <1km。

   按区段:児島→岡山 10 处(最多)、伊予大洲→内子 5、旭川→上川 3,详见 `route_break_audit_connector_chords.csv`(每处含经纬度、纯铁轨绕行长度、ratio、Google Maps 链接)。

## 三、修复建议(待你确认)

连接边的本意是"站内换乘 / 补 N02 漏掉的真实股道连接",**不该当作行车通过边去和真实铁轨竞争**。三种改法:

1. **(推荐) 给连接边也计惩罚 / 加固定附加费**,让它只在"真没有铁轨可走"时才被选中。
   - 改动小(就 4215–4217 一带):去掉 `is_station_connector` 的豁免,或给连接边加一个大的固定 surcharge。
   - A/B 已证明:这能让上述 18 班回到真实铁轨上,长度几乎不变、不绕路;真正需要连接边的地方(无平行铁轨)它仍是唯一选择,照常被选。

2. **调小 `STATION_TRANSFER_MAX_NODE_GAP_METERS`** (900 → ~150):直接砍掉长弦,但可能误伤真实枢纽里 200–400m 的合理连接。

3. **禁止连接边作为通过边**(仅在区段首末用于站点吸附):最干净但改动最大。

**安全性**:A/B 只覆盖了用到连接弦的 18 班。落地修复前应**对全部 67 班用新规则重算并重新审计一遍**,确认其余 49 班不依赖连接边(尤其那 5 处大抄近路和 2 处 >30km 的点,需确认改后仍能正常连通)。
